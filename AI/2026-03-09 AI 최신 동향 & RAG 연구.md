## 2. RAG 기술 심층 분석

### Basic RAG
- 쿼리 → 벡터 DB 1회 검색 → LLM 프롬프트에 맥락 주입 → 답변 생성
- 최신 정보 반영 + 출처 명시 가능
- **한계**: 문서가 길면 문맥 단절, 단순 키워드 매칭으로 의도 불일치 검색
- 📄 *A Comprehensive Survey of RAG* (2410.12837) - RAG의 기본 아키텍처와 진화 방향 정리

### Advanced RAG
- **Semantic Chunking**: 의미 단위 문서 분할 + Overlap으로 문맥 보존
- **HyDE** (Hypothetical Document Embeddings): 가상 답변을 먼저 생성해 검색 정확도 향상
- **Multi-query Expansion**: 쿼리를 다양하게 변형하여 넓은 범위 검색
- **Contextual Retrieval**: 문서의 전역 맥락을 개별 청크에 주입
- **Reranking**: 크로스 인코더로 검색 결과 관련성 재평가, 최적 정보만 LLM에 전달
- **Contrastive ICL**: 대조적 인컨텍스트 학습으로 품질 향상
- 📄 *Enhancing RAG: Best Practices* (2501.07391) - 쿼리 확장, 리랭킹, 최적 청크 사이즈 등 모범 사례

### Agentic RAG
- 에이전트가 **자율적으로 검색 과정을 통제**
- LLM이 어떤 DB(사내 문서 vs 일반 지식)를 검색할지 스스로 결정
- 정보 부족 시 웹 검색 도구 호출 또는 검색어 수정 후 재검색 (**자기 반성 루프**)
- 정보 없을 경우 안전망(Failsafe) 경로로 환각 원천 차단
- **Planning + Reflection + Tool Use** 패턴의 결합
- 📄 *Agentic RAG Survey* (2501.09136) - 에이전틱 RAG 설계 패턴, 다중 에이전트 협업 아키텍처, 의료/금융/교육 적용 사례
- 🎬 *What is Agentic RAG?* (IBM, 308K), *Every RAG Strategy Explained* (Cole Medin)

### RAG 평가 방법론
- Precision@K, Recall@K로 검색 품질 측정
- 📄 *RAG Evaluation Survey* (2504.14891) - 전통적/신흥 평가 방법, 사실 정확도, 안전성, 계산 효율 종합 리뷰

## 3. AI Agent 아키텍처 & 멀티에이전트 시스템

### 에이전트 진화 단계
| 유형                 | 설명                               |
| ------------------ | -------------------------------- |
| Simple Reflex      | If-Then 규칙으로만 반응                 |
| Model-based Reflex | 과거 상태/행동 영향을 기억하는 내부 모델 보유       |
| Goal-based         | 목표 설정 → 미래 시뮬레이션 → 의사결정          |
| Utility-based      | 여러 경로 중 최적(가장 빠르고 효율적) 경로 선택     |
| Learning           | 환경 피드백(보상)으로 스스로 성능 평가 및 행동 업데이트 |

### LLM 에이전트 핵심 구성요소
인지(Perception) → 뇌(Brain/LLM) → 계획(Planning) → 행동(Action) → 도구 사용(Tool Use)

### 멀티에이전트 협업 메커니즘
- **수평적 P2P**: 에이전트 간 동등한 위치에서 협력
- **계층형 (Orchestrator 중심)**: 오케스트레이터가 서브 에이전트(Planner, Worker, Critic)에게 작업 지시
- 📄 *Multi-Agent Collaboration Mechanisms* (2501.06322) - 통신 프로토콜 및 협업/경쟁 구조 설계
- 📄 *Agentic LLMs Survey* (2503.23037) - Reasoning, Action, Interaction 3축 분류
- 📄 *Agentic AI Architectures & Taxonomies* (2601.12560) - 아키텍처 분류 체계 및 평가 프레임워크

## 4. Test-Time Compute 스케일링 핵심 발견

| 발견 | 내용 |
|------|------|
| 모델 크기의 대안 | 추론 시 컴퓨팅 확장이 파라미터 확장보다 효과적일 수 있음. 소형 모델이 14배 큰 모델 능가 |
| 적응적 할당 | 난이도별 연산량 유동 조절 → Best-of-N 대비 4배 효율 개선 |
| 절대적 승자 없음 | 모든 태스크에 보편적으로 최적인 단일 기법은 없음. 모델 유형/난이도에 따라 최적 전략 상이 |
| Short vs Long horizon | 짧은 탐색과 긴 탐색 간 명확한 트레이드오프 존재 |

- 📄 *Scaling LLM Test-Time Compute* (2408.03314)
- 📄 *Inference-Time Computations Benchmark* (2502.12521) - 11개+ 추론/계획 태스크 분석
- 📄 *The Art of Scaling Test-Time Compute* (2512.02008) - 30B+ 토큰, 8개 오픈소스 LLM(7B~235B) 실험

## 6. 학습 로드맵

### 초급: 프롬프트 엔지니어링 & 기반 다지기
- AI를 검색기가 아닌 **예측 엔진**으로 이해
- 역할(Persona) + 문맥(Context) + 제약 조건 + Few-shot 예시로 프롬프트 구조화 연습
- 📄 읽기: *A Comprehensive Survey of RAG* (2410.12837) - 기본 아키텍처와 개념 이해
- 🎬 보기: *Google's 9hr Prompt Engineering in 20min* (Tina Huang), *You SUCK at Prompting AI* (NetworkChuck)

### 중급: 작업 분해, Advanced RAG & 워크플로우 구축
- [[프롬프트 체이닝]]: 복잡한 문제를 단계별 AI 호출로 분해
- [[리버스 프롬프팅]]: AI가 먼저 질문하도록 유도
- Semantic Chunking + Vector DB + Reranking 적용한 Advanced RAG 시스템 구축
- 노코드 자동화(Make, n8n)로 에이전틱 워크플로우 실습
- 📄 읽기: *Enhancing RAG: Best Practices* (2501.07391) - 쿼리 확장, 리랭킹, 최적 청크 사이즈
- 🎬 보기: *Stanford CS230 - Agents, Prompts, and RAG*, *Complete RAG Tutorial 2026* (KodeKloud)

### 고급: 에이전틱 RAG, 멀티에이전트 & 평가 시스템

- LLM-as-a-judge 기반 평가 파이프라인(Eval) 구축
- Precision@K, Recall@K 활용한 RAG 시스템 성능 평가
- Test-time compute 적응적 스케일링 구현
- 📄 읽기:
